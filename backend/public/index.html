API for Players CRUD

API Methods - Players

Operations related to players.

GET /api/players

Get all players Notice: The isActive field is present only for Elm exercises and is not included in the response for React, Redux, Vue, and VueX exercises.

Path Parameters

None

Request Body

Empty

Responses

200
Array[Player] search results matching criteria

Example Response

Content-Type: application/json

[
  {
    "id": 1,
    "name": "Gus MacGyver",
    "isActive": false
  },
  {
    "id": 2,
    "name": "Thaddeus Lynch",
    "isActive": true
  }
]

POST /api/players

Adds a new Player. Notice: id is auto-generated by the backend, do NOT add id to the request body.

Path Parameters

None

Request Body

Player Player to add.

Example Request

{
  "name": "Josianne Powlowski",
  "isActive": true
}

Responses

201
Player player created
400
Error validation error message(s)

Example Response

Content-Type: application/json

{
  "id": 3,
  "name": "Josianne Powlowski",
  "isActive": true
}

GET /api/players/{playerId}

Get a single player's information

Path Parameters

playerId
ID of the player to get

Request Body

Empty

Responses

200
Player item found
404
Player with the given ID not found

Example Response

Content-Type: application/json

{
  "id": 1,
  "name": "Gus MacGyver",
  "isActive": false
}

PUT /api/players/{playerId}

Updates a player. Notice: id is included in the URL, do NOT add id to this request body.

Path Parameters

playerId
ID of the player to be updated

Request Body

Player Player data that needs updating (not necessarily all fields)

Example Request

{
  "isActive": true
}

Responses

200
Player modified player
400
Error validation error message(s)
404
Player with the given ID not found

Example Response

Content-Type: application/json

{
    "id": 1,
    "name": "Gus MacGyver",
    "isActive": true
}

DELETE /api/players/{playerId}

Delete a player

Path Parameters

playerId
ID of the player to delete

Request Body

Empty

Responses

200
Player deleted player
404
Player with the given ID not found

Example Response

Content-Type: application/json

{
    "id": 1,
    "name": "Gus MacGyver",
    "isActive": false
}

API Methods - Authentication

Operations related to user registration

POST /api/users

Register a new user. Notice: id is auto-generated by the backend, do NOT add id to the request body. password is not included in the response.

Path Parameters

None

Request Body

User User to register.

Example Request

{
  "username": "admin",
  "password": "secret"
}

Responses

201
User registered user
400
Error validation error message(s)

Example Response

Content-Type: application/json

{
    "id": 1,
    "username": "admin"
}
